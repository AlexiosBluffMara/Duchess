# Hackathon Overview

## Competition Summary

The [**Gemma 4 Good Hackathon**](https://www.kaggle.com/competitions/gemma-4-good-hackathon) is organized by **Google DeepMind** on Kaggle. It challenges teams to build impactful applications using Google's open Gemma 4 model family.

| Detail | Value |
|:-------|:------|
| **Organizer** | Google DeepMind / Kaggle |
| **Total Prize Pool** | $200,000 |
| **Deadline** | May 18, 2026 · 11:59 PM UTC |
| **Internal Deadline** | May 7, 2026 (11 days buffer) |
| **Max Team Size** | 5 members |
| **Submissions** | 1 per team |

## Key Rules

1. **Team size**: Maximum 5 members per submission
2. **Submissions**: 1 per team
3. **External data**: Must be publicly available and free
4. **Code**: Must be open source under an OSI-approved license
5. **Winner content**: Licensed CC-BY 4.0
6. **Eligibility**: 18+, not from sanctioned countries, not a Google employee
7. **No private sharing**: Code/data cannot be shared outside team before submission
8. **Reproducibility**: Submission must be reproducible from the writeup description

[Full Rules →](https://www.kaggle.com/competitions/gemma-4-good-hackathon/rules)

## Our Strategy

**Target**: Main Track 1st ($50K) + Safety & Trust ($10K) + Digital Equity ($10K) + Cactus ($10K) + Unsloth ($10K) = **$90,000 minimum**

**Realistic ceiling**: $80,000–$100,000 across multiple tracks.

**Competitive advantages**:
1. **Real multi-device system** — glasses + phone + Mac + cloud working together (most entries will be single-device)
2. **Multimodal vision + audio** — Gemma 4 E2B's native capabilities for PPE detection and voice reporting
3. **Bilingual by design** — EN/ES in every string, every alert, every prompt
4. **Privacy-first** — Video stays on-site. WireGuard encryption. HIPAA-compliant architecture.
5. **Cactus-native design** — Our 4-tier architecture literally IS intelligent model routing
6. **Lives saved** — This isn't a toy project. 1,056 deaths/year is real.
